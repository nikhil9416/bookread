446 Chapter 11 How does the client get the stub object? The client has to get the stub object (our proxy), since that’s the thing the client will call methods on. And that’s where the RMI registry comes in. The client does a ‘lookup’, like going to the white pages of a phone book, and essentially says, “Here’s a name, and I’d like the stub that goes with that name.” Let’s take a look at the code we need to lookup and retrieve a stub object. Code Up Close MyRemote service = (MyRemote) Naming.lookup(“rmi://127.0.0.1/RemoteHello”); The client always uses the remote interface as the type of the service. In fact, the client never needs to know the actual class name of your remote service.
