Let’s take a closer look at how the AbstractClass is deﬁned, including the template method and primitive operations. abstract class AbstractClass { ﬁnal void templateMethod() { primitiveOperation1(); primitiveOperation2(); concreteOperation(); } abstract void primitiveOperation1(); abstract void primitiveOperation2(); void concreteOperation() { // implementation here } } Here’s the template method. It’s declared final to prevent subclasses from reworking the sequence of steps in the algorithm. The template method defines the sequence of steps, each represented by a method. In this example, two of the primitive operations must be implemented by concrete subclasses. We also have a concrete operation defined in the abstract class.
