leftover patterns you are here 4 627 <<interface>> Monster Prototype to the rescue The Prototype Pattern allows you to make new instances by copying existing instances. (In Java this typically means using the clone() method, or de-serialization when you need deep copies.) A key aspect of this pattern is that the client code can make new instances without knowing which speciﬁc class is being instantiated. Prototype Beneﬁts ß Hides the complexities of making new instances from the client. ß Provides the option for the client to generate objects whose type is not known. ß In some circumstances, copying an object can be more efficient than creating a new object. ß Prototype should be considered when a system must create new objects of many types in a complex class hierarchy. ß A drawback to using the Prototype is that making a copy of an object can sometimes be complicated.
