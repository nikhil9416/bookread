542 Chapter 12 Now for the Controller It’s time to write the missing piece: the controller. Remember the controller is the strategy that we plug into the view to give it some smarts. Because we are implementing the Strategy Pattern, we need to start with an interface for any Strategy that might be plugged into the DJ View. We’re going to call it ControllerInterface. public interface ControllerInterface { void start(); void stop(); void increaseBPM(); void decreaseBPM(); void setBPM(int bpm); } Here are all the methods the view can call on the controller. These should look familiar after seeing the model’s interface. You can stop and start the beat generation and change the BPM. This interface is “richer” than the BeatModel interface because you can adjust the BPMs with increase and decrease. You’ve seen that the view and controller together make use of the Strategy Pattern.
