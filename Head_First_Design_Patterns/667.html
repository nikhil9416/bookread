leftover patterns you are here 4 629 The Visitor drops by The Visitor must visit each element of the Composite; that functionality is in a Traverser object. The Visitor is guided by the Traverser to gather state from all of the objects in the Composite. Once state has been gathered, the Client can have the Visitor perform various operations on the state. When new functionality is required, only the Visitor must be enhanced. Visitor Beneﬁts ß Allows you to add operations to a Composite structure without changing the structure itself. ß Adding new operations is relatively easy. ß The code for operations performed by the Visitor is centralized. ß The Composite classes’ encapsulation is broken when the Visitor is used. ß Because the traversal function is involved, changes to the Composite structure are more difficult.
