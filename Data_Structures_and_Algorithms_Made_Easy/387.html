Since the time complexity of both the insertion algorithm and deletion algorithm is O(logn) (where n is the number of items in the heap), the time complexity of the heap sort algorithm is O(nlogn). 7.8 Priority Queues [Heaps]: Problems & Solutions Problem-1 What are the minimum and maximum number of elements in a heap of height h? Solution: Since heap is a complete binary tree (all levels contain full nodes except possibly the lowest level), it has at most 2 h+1 – 1 elements (if it is complete). This is because, to get maximum nodes, we need to fill all the h levels completely and the maximum number of nodes is nothing but the sum of all nodes at all h levels. To get minimum nodes, we should fill the h – 1 levels fully and the last level with only one element. As a result, the minimum number of nodes is nothing but the sum of all nodes from h –1 levels plus 1 (for the last level) and we get 2 h –1+1=2 h elements (if the lowest level has just 1 element and all the other levels are complete).
