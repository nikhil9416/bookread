Note that if the greedy algorithm creates a new room for the current class c i , then because it examines classes in order of start times, c i start point must intersect with the last class in all of the current rooms. Thus when greedy creates the last room, n, it is because the start time of the current class intersects with n – 1 other classes. Problem-10 Suppose we are given two arrays Start [1 ..n] and Finish[1 ..n] listing the start and finish times of each class. Our task is to choose the largest possible subset X ∈ {1,2,...,n} so that for any pair i,j ∈ X, either Start [i ] > Finish[j ] or Start [j ]> Finish [i ] Solution: Our aim is to finish the first class as early as possible, because that leaves us with the most remaining classes. We scan through the classes in order of finish time, and whenever we encounter a class that doesn’t conflict with the latest class so far, then we take that class. This algorithm clearly runs in O(nlogn) time due to sorting.
