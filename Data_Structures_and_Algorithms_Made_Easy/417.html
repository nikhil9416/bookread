MAKESET FIND UNION 8.9 Fast UNION Implementations (Quick FIND) The main problem with the previous approach is that, in the worst case we are getting the skew trees and as a result the FIND operation is taking O(n) time complexity. There are two ways to improve it: • UNION by Size (also called UNION by Weight): Make the smaller tree a subtree of the larger tree • UNION by Height (also called UNION by Rank): Make the tree with less height a subtree of the tree with more height
