75 Hash tables Your buddy Maggie can give you the price in O(1) time for any item, no matter how big the book is. She’s even faster than binary search. What a wonderful person! How do you get a “Maggie”? Let’s put on our data structure hats. You know two data structures so far: arrays and lists (I won’t talk about stacks because you can’t really “search” for something in a stack). You could implement this book as an array. Each item in the array is really two items: one is the name of a kind of produce, and the other is the price. If you sort this array by name, you can run binary search on it to ﬁnd the price of an item. So you can ﬁnd items in O(log n) time. But you want to ﬁnd items in O(1) time. at is, you want to make a “Maggie.” at’s where hash functions come in.
