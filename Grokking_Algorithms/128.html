109 Implementing the algorithm And so on. e algorithm will keep going until either • A mango seller is found, or • e queue becomes empty, in which case there is no mango seller. Alice and Bob share a friend: Peggy. So Peggy will be added to the queue twice: once when you add Alice’s friends, and again when you add Bob’s friends. You’ll end up with two Peggys in the search queue. But you only need to check Peggy once to see whether she’s a mango seller. If you check her twice, you’re doing unnecessary, extra work. So once you search a person, you should mark that person as searched and not search them again. If you don’t do this, you could also end up in an inﬁnite loop.
