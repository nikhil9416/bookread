116 Chapter 7 I Dijkstra’s algorithm In the last chapter, you ﬁgured out a way to get from point A to point B. It’s not necessarily the fastest path. It’s the shortest path, because it has the least number of segments (three segments). But suppose you add travel times to those segments. Now you see that there’s a faster path. You used breadth-ﬁrst search in the last chapter. Breadth-ﬁrst search will ﬁnd you the path with the fewest segments (the ﬁrst graph shown here). What if you want the fastest path instead (the second graph)? You can do that fastest with a diﬀerent algorithm called Dijkstra’s algorithm. Working with Dijkstra’s algorithm Let’s see how it works with this graph. Each segment has a travel time in minutes.
