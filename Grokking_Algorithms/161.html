142 Chapter 8 I Greedy algorithms The classroom scheduling problem Suppose you have a classroom and want to hold as many classes here as possible. You get a list of classes. You can’t hold all of these classes in there, because some of them overlap. You want to hold as many classes as possible in this classroom. How do you pick what set of classes to hold, so that you get the biggest set of classes possible? Sounds like a hard problem, right? Actually, the algorithm is so easy, it might surprise you. Here’s how it works: 1. Pick the class that ends the soonest. is is the ﬁrst class you’ll hold in this classroom. 2. Now, you have to pick a class that starts aﬅer the ﬁrst class. Again, pick the class that ends the soonest. is is the second class you’ll hold.
