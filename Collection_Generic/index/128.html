The conversion from List to List<Integer> of the list returned by getItems does issue an unchecked warning. At run-time, a class cast exception will be raised when attempting to extract data from these lists, since the cast to type Integer implicitly inserted by erasure will fail. (The failure of these casts does not constitute a violation of the cast-iron guarantee, because this guarantee doesn’t hold in the presence of legacy code or unchecked warnings.) Because the exception is raised far from the place where the strings are added to the lists, the bug may be hard to pinpoint. If the legacy library has been generified by applying the minimal changes or stubs tech- niques (see Sections Section 5.4.1 and Section 5.4.2), then these problems cannot arise as long as generic types have been assigned correctly.
