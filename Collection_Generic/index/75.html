} } We intend this code to work as follows: assert allZero(Arrays.asList(0,0,0)); assert allZero(Arrays.asList("","","")); However, in this case the erasures of the signatures of both methods are identical: boolean allZero(List) Therefore, a name clash is reported at compile time. It is not possible to give both methods the same name and try to distinguish between them by overloading, because after erasure it is impossible to distinguish one method call from the other. For another example, here is a bad version of the integer class, that tries to make it possible to compare an integer with either an integer or a long: class Integer implements Comparable<Integer>, Comparable<Long> { // compile-time error, cannot implement two interfaces with same erasure private final int value; public Integer(int value) { this.value = value; } public int compareTo(Integer i) { return (value < i.value) ? -1 : (value == i.value) ? 0 : 1; } public int compareTo(Long l) { return (value < l.value) ? -1 : (value == l.value) ? 0 : 1; } } If this were supported, it would, in general, require a complex and confusing definition of bridge methods (see Section 3.7).
