(See [CPJ 2.4.4] for more information on copy-on-write.) 5.3 Blocking queues and the producer-consumer pattern Blocking queues provide blocking put and take methods as well as the timed equivalents offer and poll. If the queue is full, put blocks until space becomes available; if the queue is empty, take blocks until an element is available. Queues can be bounded or unbounded; unbounded queues are never full, so a put on an unbounded queue never blocks. Blocking queues support the producer-consumer design pattern. A producer- consumer design separates the identiﬁcation of work to be done from the exe- cution of that work by placing work items on a “to do” list for later processing, rather than processing them immediately as they are identiﬁed.
