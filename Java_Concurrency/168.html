If get terminates with a TimeoutException, the task is cancelled via its Future. If the underlying computation throws an exception prior to cancellation, it is rethrown from timed- Run, which is the most convenient way for the caller to deal with the exception. When Future.get throws InterruptedException or TimeoutExcep- tion and you know that the result is no longer needed by the program, cancel the task with Future.cancel. 7.1.6 Dealing with non-interruptible blocking Many blocking library methods respond to interruption by returning early and throwing InterruptedException, which makes it easier to build tasks that are responsive to cancellation. However, not all blocking methods or blocking mech- anisms are responsive to interruption; if a thread is blocked performing syn- chronous socket I/O or waiting to acquire an intrinsic lock, interruption has no effect other than setting the threadâ€™s interrupted status. We can sometimes con- vince threads blocked in noninterruptible activities to stop by means similar to interruption, but this requires greater awareness of why the thread is blocked.
