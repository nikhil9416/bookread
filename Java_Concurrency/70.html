This combination of an immutable holder object for multiple state variables related by an invariant, and a volatile reference used to ensure its timely visibility, allows VolatileCachedFactorizer to be thread-safe even though it does no explicit locking. 3.5 Safe publication So far we have focused on ensuring that an object not be published, such as when it is supposed to be conﬁned to a thread or within another object. Of course, sometimes we do want to share objects across threads, and in this case we must do so safely.
