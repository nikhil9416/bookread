Summary Explicit Locks offer an extended feature set compared to intrinsic locking, includ- ing greater ﬂexibility in dealing with lock unavailability and greater control over queueing behavior. But ReentrantLock is not a blanket substitute for synchro- nized; use it only when you need features that synchronized lacks. Read-write locks allow multiple readers to access a guarded object concur- rently, offering the potential for improved scalability when accessing read-mostly data structures.
