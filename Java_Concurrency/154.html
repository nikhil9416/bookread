6.3. Finding exploitable parallelism 133 out which. Summary Structuring applications around the execution of tasks can simplify development and facilitate concurrency. The Executor framework permits you to decouple task submission from execution policy and supports a rich variety of execution policies; whenever you ﬁnd yourself creating threads to perform tasks, consider using an Executor instead. To maximize the beneﬁt of decomposing an applica- tion into tasks, you must identify sensible task boundaries. In some applications, the obvious task boundaries work well, whereas in others some analysis may be required to uncover ﬁner-grained exploitable parallelism.
