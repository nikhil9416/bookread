Chapter 15 Atomic Variables and Nonblocking Synchronization Many of the classes in java.util.concurrent, such as Semaphore and Concur- rentLinkedQueue, provide better performance and scalability than alternatives using synchronized. In this chapter, we take a look at the primary source of this performance boost: atomic variables and nonblocking synchronization. Much of the recent research on concurrent algorithms has focused on nonblock- ing algorithms, which use low-level atomic machine instructions such as compare- and-swap instead of locks to ensure data integrity under concurrent access. Non- blocking algorithms are used extensively in operating systems and JVMs for thread and process scheduling, garbage collection, and to implement locks and other concurrent data structures. Nonblocking algorithms are considerably more complicated to design and im- plement than lock-based alternatives, but they can offer signiﬁcant scalability and liveness advantages. They coordinate at a ﬁner level of granularity and can greatly reduce scheduling overhead because they don’t block when multiple threads con- tend for the same data. Further, they are immune to deadlock and other liveness problems. In lock-based algorithms, other threads cannot make progress if a thread goes to sleep or spins while holding a lock, whereas nonblocking algo- rithms are impervious to individual thread failures. As of Java 5.0, it is possible to build efﬁcient nonblocking algorithms in Java using the atomic variable classes such as AtomicInteger and AtomicReference. Atomic variables can also be used as “better volatile variables” even if you are not developing nonblocking algorithms.
