Same lambda, different functional interfaces Because of the idea of target typing, the same lambda expression can be associated with different functional interfaces if they have a compatible abstract method signature. For example, both interfaces Callable and PrivilegedAction described earlier represent functions that accept nothing and return a generic type T. The following two assignments are therefore valid: Callable<Integer> c = () -> 42; PrivilegedAction<Integer> p = () -> 42; In this case the first assignment has target type Callable<Integer> and the second assignment has target type PrivilegedAction<Integer>.
