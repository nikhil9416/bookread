A.1.1. Repeated annotations Previous versions of Java forbid more than one annotation of a given annotation type to be specified on a declaration. For this reason, the following code is invalid: Java EE programmers often make use of an idiom to circumvent this restriction. You declare a new annotation, which contains an array of the annotation you want to repeat. It looks like this: @interface Author { String name(); } @interface Authors { Author[] value(); } @Authors( { @Author(name="Raoul"), @Author(name="Mario") , @Author(name="Alan")} ) class Book{} The nested annotation on the Book class is pretty ugly. This is why Java 8 essentially removes this restriction, which tidies things a bit. You’re now allowed to specify multiple annotations of the same annotation type on a declaration, provided they stipulate that the annotation is repeatable. It’s not the default behavior; you have to explicitly ask for an annotation to be repeatable. www.it-ebooks.info
