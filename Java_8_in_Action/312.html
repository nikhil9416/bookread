Using a Future to execute a long operation asynchronously Can you think of a problem with this scenario? What if the long operation never returns? Futures limitations This first small example shows that the Future interface provides methods to check if the asynchronous computation is complete (using the isDone method), to wait for its completion, and to retrieve its result. But these features aren’t enough to let you write concise concurrent code.
