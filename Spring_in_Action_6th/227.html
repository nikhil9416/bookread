So, how can we use the access token? Presumably, we’ll send it in a request to the Taco Cloud API as part of the "Authorization" header—perhaps something like this: $ curl localhost:8080/ingredients \ -H"Content-type: application/json" \ -H"Authorization: Bearer eyJraWQ..." \ -d'{"id":"FISH","name":"Stinky Fish", "type":"PROTEIN"}' At this point, the token achieves nothing for us. That’s because our Taco Cloud API hasn’t been enabled to be a resource server yet. But in lieu of an actual resource server and client API, we can still inspect the access token by copying it and pasting into the form at https://jwt.io.
