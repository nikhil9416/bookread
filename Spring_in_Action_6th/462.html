435 Monitoring Spring with JMX For over a decade and a half, Java Management Extensions (JMX) has been the standard means of monitoring and managing Java applications. By exposing man- aged components known as MBeans (managed beans), an external JMX client can manage an application by invoking operations, inspecting properties, and monitor- ing events from MBeans. We’ll start exploring Spring and JMX by looking at how Actuator endpoints are exposed as MBeans. 17.1 Working with Actuator MBeans By default, all Actuator endpoints are exposed as MBeans. But, starting with Spring Boot 2.2, JMX itself is disabled by default. To enable JMX in your Spring Boot appli- cation, you can set spring.jmx.enabled to true.
