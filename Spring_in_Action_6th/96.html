69 Reading and writing data with JDBC @Override public Ingredient save(Ingredient ingredient) { jdbcTemplate.update( "insert into Ingredient (id, name, type) values (?, ?, ?)", ingredient.getId(), ingredient.getName(), ingredient.getType().toString()); return ingredient; } Because it isn’t necessary to map ResultSet data to an object, the update() method is much simpler than query(). It requires only a String containing the SQL to perform as well as values to assign to any query parameters. In this case, the query has three parameters, which correspond to the final three parameters of the save() method, providing the ingredient’s ID, name, and type. With JdbcIngredientRepository complete, you can now inject it into Design- TacoController and use it to provide a list of Ingredient objects instead of using hardcoded values (as you did in chapter 2).
