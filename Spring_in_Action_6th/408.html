As with the request-stream model, the Flux returned enables the controller to stream multiple values to the client. But the Flux parameter is what differentiates the chan- nel model from the request-stream model. The Flux parameter coming in allows the controller to handle a stream of requests from the client coming into the handler method. The client side of the channel model differs from the client of the request- stream model only in that it sends a Flux<GratuityIn> to the server instead of a Mono<GratuityIn>, as shown here.
