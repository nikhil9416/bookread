477 Building and running projects You may also notice that many of the spring init parameters are the same as or simi- lar to the parameters for the curl option. That said, the spring init command doesn’t support all of the same parameters as the curl option (e.g., baseDir), and the parameters are hyphen-delimited instead of camelCase (e.g., package-name versus packageName). A.6 Building and running projects No matter how you initialize your project, you can always run the application from the command line with the java -jar command as follows: % java -jar demo.jar This will even work if you decide to create a WAR file distribution instead of a JAR file, as shown here: % java -jar demo.war You can also take advantage of the Spring Boot Maven and Gradle plugins to run your application. For example, if your project is built with Maven, you can run it like this: % mvn spring-boot:run If, on the other hand, you’ve chosen to build your project with Gradle, you can run your project like this: % gradle bootRun In either case, whether using Maven or Gradle, the build tool will first build your proj- ect (if it hasn’t already been built) and run it.
