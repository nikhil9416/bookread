473 Initializing a project from the command line You can consume the API in the following two ways: ■ Using the curl command (or some similar command-line REST client command) ■ Using the Spring Boot command-line interface (aka, Spring Boot CLI) Let’s look at these options, starting with the curl command. curl and the Initializr API The simplest way to bootstrap a Spring project with curl is to consume the API like this: % curl https://start.spring.io/starter.zip -o demo.zip In this case, you’re requesting the /starter.zip endpoint from the Initializr, which will generate a Spring project and download it as a zip file.
