109 Using GORM for data persistence The same dependency can be expressed in a Gradle build like this: compile("org.grails:gorm-hibernate4-spring-boot:1.1.0.RELEASE") This library carries some Spring Boot auto-configuration with it that will automatically configure all of the necessary beans to support working with GORM. All you need to do is start writing the code. Due to the nature of how GORM works, it requires that at least the entity class be writ- ten in Groovy. We’ve already written the Book entity in listing 6.1. For many projects, this will be fine. If, how- ever, you’re interested in working with the MongoDB document database, you’ll be pleased to know that GORM for MongoDB is also available for Spring Boot.
