But user John isn’t allowed to call path /a/b, so authenticating with his credentials for this call generates a 403 Forbidden: curl -u john:12345 -XGET http://localhost:8080/a/b The response is { "status":403, "error":"Forbidden", "message":"Forbidden", "path":"/a/b" } With this example, you now know how to differentiate requests based on the HTTP method. But, what if multiple paths have the same authorization rules? Of course, we can enumerate all the paths for which we apply authorization rules, but if we have too many paths, this makes reading code uncomfortable. As well, we might know from the beginning that a group of paths with the same prefix always has the same authoriza- tion rules. We want to make sure that if a developer adds a new path to the same group, it doesn’t also change the authorization configuration. To manage these cases, we use path expressions. Let’s prove these in an example. For the current project, we want to ensure that the same rules apply for all requests for paths starting with /a/b. These paths in our case are /a/b and /a/b/c. To achieve this, we use the ** operator. (Spring MVC borrows the path-matching syntaxes from Ant.)
