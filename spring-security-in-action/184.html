154 CHAPTER 7 Configuring authorization: Restricting access most applications, it doesn’t happen that all the users identified by the system can access every resource in the system. In this chapter, we’ll discuss authorization. Autho- rization is the process during which the system decides if an identified client has per- mission to access the requested resource (figure 7.1). Figure 7.1 Authorization is the process during which the application decides whether an authenticated entity is allowed to access a resource. Authorization always happens after authentication. In Spring Security, once the application ends the authentication flow, it delegates the request to an authorization filter. The filter allows or rejects the request based on the configured authorization rules (figure 7.2). To cover all the essential details on authorization, in this chapter we’ll follow these steps: 1 Gain an understanding of what an authority is and apply access rules on all endpoints based on a user’s authorities. 2 Learn how to group authorities in roles and how to apply authorization rules based on a user’s roles.
