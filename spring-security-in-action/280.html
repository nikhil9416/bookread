250 CHAPTER 11 Hands-on: A separation of responsibilities At the implementation level, tokens can even be regular strings. What’s most impor- tant is to be able to recognize these after you issue them. You can generate UUIDs and store them in memory or in a database. Let’s assume the following scenario: 1 The client proves its identity to the server with its credentials. 2 The server issues the client a token in the format of a UUID. This token, now associated with the client, is stored in memory by the server (figure 11.6). Figure 11.6 When the client authenticates, the server generates a token and returns it to the client. This token is then used by the client to access resources on the server. 3 When the client calls an endpoint, the client provides the token and gets autho- rized. Figure 11.7 presents this step.
