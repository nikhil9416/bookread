151 Running and testing the application 6.5 Running and testing the application We have finished writing the code for the first hands-on project of the book. It is time to verify that it is working according to our specifications. So let’s run the application and try to log in. After running the appli- cation, we can access it in the browser by typing the address http://localhost :8080. The standard login form appears as presented in figure 6.4. The user I stored in the database (and the one in the script given at the beginning of this chap- ter) is John with the password 12345 that’s hashed using bcrypt. You can use these cre- dentials to log in. NOTE In a real-world application, you should never allow your users to define simple passwords like “12345.” Passwords so simple are easy to guess and rep- resent a security risk. Wikipedia provides an informative article on passwords at https://en.wikipedia.org/wiki/Password_strength. It explains not only the rules to set strong passwords, but also how to calculate password strength. Once logged in, the application redirects you to the main page (figure 6.5). Here, the username taken from the security context appears on the page, together with the list of the products from the database. When you click the Sign Out Here link, the application redirects you to the stan- dard sign out confirmation page (figure 6.6). This is predefined by Spring Security because we use the formLogin authentication method.
