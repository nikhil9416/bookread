The input data provided to the sorting algorithms can be preprocessed to ensure some of the following properties (not all are compatible): Sorted The input elements can be presorted into ascending order (the ultimate goal) or in descending order. Killer median-of-three Musser (1997) discovered an ordering that ensures that Quicksort requires O(n 2 ) comparisons when using median-of-three to choose a pivot. Nearly sorted Given a set of sorted data, we can select k pairs of elements to swap and the distance d with which to swap (or 0 if any two pairs can be swapped). Using this capability, you can construct input sets that might more closely match your input set.
