Example 6-2. Breadth-First Search implementation /** * Perform breadth-first search on graph from vertex s, and compute BFS * distance and pred vertex for all vertices in the graph. */ void bfsSearch (Graph const &graph, int s, /* in */ vector<int> &dist, vector<int> &pred) /* out */ { // initialize dist and pred to mark vertices as unvisited. Begin at s // and mark as Gray since we haven't yet visited its neighbors. const int n = graph.numVertices(); pred.assign(n, −1); dist.assign(n, numeric_limits<int>::max()); vector<vertexColor> color (n, White); dist[s] = 0; color[s] = Gray; queue<int> q; q.push(s); while (!q.empty()) { int u = q.front(); // Explore neighbors of u to expand the search horizon for (VertexList::const_iterator ci = graph.begin (u); ci != graph.end (u); ++ci) { int v = ci->first; if (color[v] == White) { dist[v] = dist[u]+1; pred[v] = u; color[v] = Gray; q.push(v); } } q.pop(); color[u] = Black; } } Analysis During initialization, Breadth-First Search updates information for all vertices, with performance O(V). When a vertex is first visited (and colored gray), it is inser‐ ted into the queue, and no vertex is added twice. Since the queue can add and remove elements in constant time, the cost of managing the queue is O(V). Finally, each vertex is dequeued exactly once and its adjacent vertices are traversed exactly once. The sum total of the edge loops, therefore, is bounded by the total number of edges, or O(E). Thus, the total performance is O(V + E). 146 | Chapter 6: Graph Algorithms
