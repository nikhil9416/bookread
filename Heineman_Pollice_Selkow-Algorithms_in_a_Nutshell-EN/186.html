The IPlayer interface abstracts the abilities of a player to manipulate the game state. It defines how to: Evaluate a board eval(IGameState) returns an integer evaluating the game state from the play‐ er’s perspective; score(IGameScore) sets the scoring computation the player uses to evaluate a game state. Generate valid moves validMoves(IGameState) returns a collection of available moves given the game state. The IGameMove interface defines how moves manipulate the game state. The move classes are problem-specific, and the search algorithm need not be aware of their specific implementation. IGameScore defines the interface for scoring states. From a programming perspective, the heart of the path-finding algorithm for a game tree is an implementation of the IEvaluation interface shown in Example 7-1. Example 7-1.
