Dijkstra’s Algorithm example Dijkstra’s Algorithm conceptually operates in a greedy fashion by expanding a set of vertices, S, for which the shortest path from a designated source vertex, s, to every vertex v ∈ S is known, but only using paths that include vertices in S. Initially, S equals the set {s}. To expand S, as shown in Figure 6-12, Dijkstra’s Algorithm finds the vertex v ∈ V – S (i.e., the vertices outside the shaded region) whose distance to s is smallest, and follows v’s edges to see whether a shorter path exists to some other vertex. After processing v 2 , for example, the algorithm determines that the distance from s to v 3 containing only vertices in S is really 17 through the path <s, v 2 , v 3 >. Once S equals V, the algorithm completes and the final result is depicted in Figure 6-12. Input/Output The input is a directed, weighted graph G = (V, E) and a source vertex s ∈ V.
