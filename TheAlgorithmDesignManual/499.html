15.4 SHORTEST PATH 489 INPUT OUTPUT 15.4 Shortest Path Input description: An edge-weighted graph G, with vertices s and t. Problem description: Find the shortest path from s to t in G. Discussion: The problem of ﬁnding shortest paths in a graph has several applica- tions, some quite surprising: • The most obvious applications arise in transportation or communications, such as ﬁnding the best route to drive between Chicago and Phoenix or ﬁguring how to direct packets to a destination across a network. • Consider the task of partitioning a digitized image into regions containing distinct objects—a problem known as image segmentation. Separating lines are needed to carve the space into regions, but what path should these lines take through the grid? We may want a line that relatively directly goes from x to y, but avoids cutting through object pixels as much as possible. This grid of pixels can be modeled as a graph, with the cost of an edge reﬂecting the color transitions between neighboring pixels. The shortest path from x to y in this weighted graph deﬁnes the best separating line. • Homophones are words that sound alike, such as to, two, and too. Distinguish- ing between homophones is a major problem in speech recognition systems.
