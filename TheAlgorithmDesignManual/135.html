Quicksort (shown in action in Figure 4.5) separates the n − 1 other items into two piles: a low pile containing all the elements that appear before p in sorted order and a high pile containing all the elements that appear after p in sorted order. Low and high denote the array positions we place the respective piles, leaving a single slot between them for p. Such partitioning buys us two things. First, the pivot element p ends up in the exact array position it will reside in the the ﬁnal sorted order. Second, after partitioning no element ﬂops to the other side in the ﬁnal sorted order. Thus we can now sort the elements to the left and the right of the pivot independently! This gives us a recursive sorting algorithm, since we can use the partitioning approach to sort each subproblem.
