I encourage you to analyze these implemen- tations carefully to see where the problematic cases arise and why. 5.9.1 Finding Cycles Back edges are the key to ﬁnding a cycle in an undirected graph. If there is no back edge, all edges are tree edges, and no cycle exists in a tree. But any back edge going from x to an ancestor y creates a cycle with the tree path from y to x. Such a cycle is easy to ﬁnd using dfs: process_edge(int x, int y) { if (parent[x] != y) { /* found back edge! */ printf("Cycle from %d to %d:",y,x); find_path(y,x,parent); printf("\n\n"); finished = TRUE; } } The correctness of this cycle detection algorithm depends upon processing each undirected edge exactly once. Otherwise, a spurious two-vertex cycle (x, y, x) could be composed from the two traversals of any single undirected edge. We use the finished ﬂag to terminate after ﬁnding the ﬁrst cycle. 5.9.2 Articulation Vertices Suppose you are a vandal seeking to disrupt the telephone network.
