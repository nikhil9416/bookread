At each iteration, we add one new vertex into the spanning tree. A greedy algorithm suﬃces for correctness: we always add the lowest-weight edge linking a vertex in the tree to a vertex on the outside. The simplest implementation of this idea would assign each vertex a Boolean variable denoting whether it is already in the tree (the array intree in the code below), and then searches all edges at each iteration to ﬁnd the minimum weight edge with exactly one intree vertex. Our implementation is somewhat smarter. It keeps track of the cheapest edge linking every nontree vertex in the tree. The cheapest such edge over all remaining non-tree vertices gets added in each iteration. We must update the costs of getting to the non-tree vertices after each insertion.
