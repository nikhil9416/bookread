Note that F (4) is computed on both sides of the recursion tree, and F (2) is computed no less than ﬁve times in this small example. The weight of all this redundancy becomes clear when you run the program. It took more than 7 minutes for my program to compute the ﬁrst 45 Fibonacci numbers. You could probably do it faster by hand using the right algorithm. How much time does this algorithm take to compute F (n)? Since F n+1 /F n ≈ φ = (1 + √ 5)/2 ≈ 1.61803, this means that F n > 1.6 n . Since our recursion tree has only 0 and 1 as leaves, summing up to such a large number means we must have at least 1.6 n leaves or procedure calls! This humble little program takes exponential time to run! 8.1.2 Fibonacci Numbers by Caching In fact, we can do much better. We can explicitly store (or cache) the results of each Fibonacci computation F (k) in a table data structure indexed by the parameter k.
